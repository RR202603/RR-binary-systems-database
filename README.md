# 二元系 TDB 静态检索站点（示例项目）

> 目标：像截图那样在网页中点选两个元素，立刻出现对应二元系的 TDB 文件并可下载。

## 目录结构

```
phaseweb/
├─ index.html              # 页面
├─ styles.css              # 页面样式
├─ script.js               # 逻辑（渲染元素周期表、筛选）
├─ data/
│  └─ systems.json         # 由脚本生成的索引（元素对 → 文件列表）
├─ tools/
│  └─ build_mapping.py     # 读取 /tdb 目录生成 systems.json 的脚本
└─ tdb/                    # 你的全部 .tdb 文件（可按子文件夹组织）
```

## 快速开始（本地预览）

1. **把你的 `.tdb` 文件都放进 `tdb/` 目录**（随便子文件夹）。建议文件名采用 `El1-El2-描述.tdb` 格式，如 `Al-Cu_Smith2019.tdb`。
2. **安装 Python 3**（Windows, macOS 或 Linux 均可）。
3. 在此目录打开终端/命令行，执行：

   ```bash
   python tools/build_mapping.py
   ```

   该脚本会扫描 `tdb/` 并生成 `data/systems.json`。

4. 启动一个本地静态服务器（浏览器直接打开 `index.html` 会有跨域限制）：

   ```bash
   python -m http.server 8000
   ```

   打开浏览器访问 `http://localhost:8000`。点击两个元素即可看到匹配到的 TDB 文件。

## 发布到 GitHub Pages（永久免费）

1. 新建 GitHub 仓库，把整个 `phaseweb/` 目录上传（包含 `tdb/` 与 `data/`）。
2. 在仓库 **Settings → Pages** 里将 **Build from** 设置为 **Deploy from a branch**，选择 `main` 分支、`/ (root)` 目录，保存。
3. 等待几分钟，GitHub 会生成一个网站地址。以后**只需把新文件放入 `tdb/` 并重新运行脚本**，提交后网页就会自动更新。

## 命名约定 & 识别规则

- 默认从**文件名**里抓取前两个出现的化学元素符号（区分大小写）。可识别：`Al-Cu.tdb`, `Cu_Al_2019.tdb`, `Ni-Ti (Smith 2004).tdb`。
- 如有 `Cu-Al` 与 `Al-Cu` 混用，脚本会自动按字母序规范化为 `Al-Cu`，统一到一个条目下。
- 如果某些文件名无法被识别（例如完全无元素符号），脚本会在终端提示“跳过”。你可以改名后再运行脚本。

## 常见问题

- **能放 1000+ 文件吗？** 可以。本项目是纯前端静态站点，`systems.json` 只是一个几百 KB 的索引文件，浏览器一次加载后在本地过滤，速度很快。
- **能否一个二元系对应多个 TDB？** 支持。右侧列表会把同一元素对下的所有文件全部列出来。
- **我不想点元素，能输入搜索吗？** 可以在 `script.js` 里很容易加入模糊搜索（为了示例简洁此处未默认开启）。
- **我不会写代码，能改风格吗？** 只需改 `styles.css`。结构和逻辑已经分离。

## 安全与隐私

- 全站纯静态，不需要数据库或后台服务器。
- 文件在你的 GitHub 仓库或你自己的服务器上，完全可控。
